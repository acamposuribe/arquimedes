## Goal

Implement `arq lint` as a two-stage maintenance system:
- deterministic health checks first
- reflective LLM passes second

The result should improve:
- cluster quality
- concept pages
- collection pages
- the memory layer

## L6.1 Deterministic Lint Core

- [ ] Create `src/arquimedes/lint.py`
- [ ] Implement `run_deterministic_lint(config)` with checks for:
  - broken links
  - orphaned materials
  - orphaned wiki pages
  - missing metadata
  - stale enrichment
  - stale index
  - stale memory bridge
  - duplicates
  - missing compiled pages
- [ ] Write `derived/lint/deterministic_report.json`
- [ ] Add `arq lint --quick`
