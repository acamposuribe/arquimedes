## Goal

Implement `arq lint` as a two-stage maintenance system:
- deterministic health checks first
- reflective LLM passes second

The result should improve:
- cluster quality
- concept pages
- collection pages
- the memory layer

Implementation order:
1. deterministic lint core
2. cluster audit
3. concept reflection
4. collection reflection
5. memory feed
6. graph reflection

Parallelism rule:
- parallelize concept reflections across independent stale clusters
- parallelize collection reflections across independent stale collections
- keep graph reflection as a final global pass

## Current Implementation Snapshot

Implemented so far in code:
- `arq lint` CLI wiring
- deterministic lint report generation
- deterministic checks for broken links, stale index/memory, missing pages, missing metadata, duplicates, orphaned materials/pages, stale enrichment
- reflective pass scaffolding for clusters, concepts, collections, and graph
- projection of reflection artifacts into SQLite memory tables
- compile-time quick lint hook
- full-lint schedule stamp
- coarse graph-reflection schedule gate

Verified in tests:
- deterministic lint
- cluster audit
- concept reflection
- collection reflection
- graph reflection
- artifact-to-wiki rendering
- quick/full scheduling behavior
